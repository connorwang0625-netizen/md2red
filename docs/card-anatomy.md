# 卡片分工

一组卡 = **封面 1 + 正文 N + 尾页 1**。三类页职责互斥，内容**不得重叠**。

## 封面（Hook）

职责：**勾人往下翻**。不剧透、不做目录、不复制尾页清单。

Swiss 结构：
- 左上标题（statement，h-xl 300）
- 巨大锚点数字（num，字重 200）+ 中文小注
- 一句**价值钩子**（点出收益，不给清单）
- 底部单行 **meta 条**（编号 · 分类 · 账号 · 时长）

唯一 issue 元素：只在顶或底放一处，**不能既顶又底**。

## 正文（一页一点）

职责：**一页讲一个要点**。内容全部在图里讲完——下方笔记文字（caption）绝大多数人不会细读，只服务算法和评论区。

Swiss 结构：
- 顶部 locator（`RITUAL 0X / 0N — 关键词`）+ 发丝线
- 轻标题（h-xl 300）
- **数据锚**（num-mega 200，锚点色）+ 标签
- `Why · 为什么`（短）
- `How · 怎么做`（短），发丝线分段
- 底部 source（出处）

密度原则：字少但信息密度高，**拇指 2 秒内结论要立得住**。别让 Why/How 涨成三四行小作文。

## 尾页（Recap + CTA）

职责：**唯一的总结页 + 行动召唤**。

Swiss 结构：
- 顶 locator
- 轻标题
- 数据 **ledger**（序号 + 关键词 + 主角数字，发丝线分行）
- 金句
- CTA（`@账号`）+ 页脚

## 反例（一眼别扭）

- 封面清单 ＝ 尾页 ledger（内容重复）→ 删封面清单，改钩子。
- 大数字做成粗体（违反「越大越细」）。
- 圆角 + 阴影卡片（违反 Swiss）。
- 一页堆成字墙。
- 间距随手取，跨页节奏漂移。
