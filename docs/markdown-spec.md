# Markdown 写法规范

一篇 = 一个 `.md` 文件（放在 `content/`）。你只写内容，封面和尾页由系统自动生成。

## 整体结构

```
--- (YAML frontmatter：元数据) ---

## 正文卡 1
... 卡内容 ...

---   ← 用三个连字符分隔下一张正文卡

## 正文卡 2
...
```

- 顶部第一段 `--- ... ---` 是 **frontmatter**（元数据），不算卡片。
- 之后每一段用单独一行的 `---` 分隔，**每段 = 一张正文卡**。
- **封面** 与 **尾页** 不用手写，由 frontmatter + 正文自动汇总生成。

## Frontmatter 字段

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `palette` | 建议 | 配色：`B` 清新薄荷 / `C` 酸性霓虹 / `D` 美拉德。默认 `B` |
| `style` | | 风格：`swiss`（默认） |
| `id` | | 编号，如 `HE-0001`，进封面与页脚 |
| `account` | | 账号，如 `玉时桂`，进尾页 CTA |
| `category` | | 分类，进封面 meta 条 |
| `title` | ✓ | 文章标题（元数据） |
| `cover_title` | ✓ | 封面大标题（钩子式短句，可用中文逗号换行）。缺省回退 `title` |
| `num` | | 封面巨大数字。缺省 = 正文卡数量 |
| `num_label` | | 数字下方中文小注，如 `个晨间仪式`。缺省回退 `category` |
| `hook` | ✓ | 封面价值钩子（一句话，不剧透、不做目录） |
| `tail_title` | | 尾页标题，缺省 `不用全做，先挑一件` |
| `closing` | | 尾页金句 |
| `cta` | | 尾页行动召唤 |

## 正文卡语法

每张正文卡（两条 `---` 之间）用这几行：

```
## 卡标题
@key 短关键词
@num 主数字|单位 · 标签说明
**为什么** 一两句论据。
**怎么做** 一两句可执行动作。
> 出处 期刊或机构名
```

| 标记 | 作用 |
| --- | --- |
| `## 标题` | 卡标题（轻标题，越大越细） |
| `@key 词` | 短关键词，用于角标和尾页 ledger。缺省回退标题 |
| `@num 主数字\|单位 · 标签` | 数据锚。`\|` 前是大数字、后是小单位；`·` 后是说明标签。单位与标签都可省略 |
| `**为什么** …` | Why 段 |
| `**怎么做** …` | How 段 |
| `> 出处 …` | 来源 |

## 自动生成

- **封面**：`cover_title` + `num`/`num_label` + `hook` + 底部 meta 条（id · 分类 · 账号）。
- **尾页**：把每张正文卡的 `@key`/`@num` 汇总成 ledger，再加 `closing` 金句与 `@account` CTA。

## 约束（只警告，不拦截）

查看器顶部会黄条提示：

- 正文卡 > 8 张（轮播建议 ≤ 8）。
- 卡标题 > 14 字、`为什么`/`怎么做` > 80 字（会变字墙）。
- 缺 `@num`、缺 `> 出处`、缺 `title`/`hook`/`cover_title`。

## 渲染与导出

1. 浏览器打开仓库根目录的 `index.html`。
2. 点「选择 .md」或把 `content/xxx.md` 拖进页面。
3. 预览每张卡；改完 MD 重新选一次即可刷新。
4. `⌘P` 打印 → 目标「另存为 PDF」或用浏览器导出，每张卡 = 一页 1080×1440。

> 用本地服务（`python3 -m http.server`）打开可自动载入 `content/HE-0001.md`；直接双击 `index.html` 则用内置示例，手动选文件即可。
